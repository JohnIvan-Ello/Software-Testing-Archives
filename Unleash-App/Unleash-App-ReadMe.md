# 🎮 Unleash App: Game Development QA
**Domain:** Game Development | Mobile & PC Gaming | Agile Quality Assurance

Unleash App is a feature-rich application centered around interactive pet customization and social gaming elements. As a QA Specialist/Lead, I was responsible for maintaining the quality of complex game features, managing the development backlog, and ensuring that all defect lifecycles were documented through professional Proof of Testing (POT) standards.

---

## 🛠️ My Role & Responsibilities
My work bridged the gap between game design and software stability. I managed the end-to-end QA workflow, ensuring that pet-specific features and system updates were verified before reaching the production environment.

### 🔍 Key QA Focus Areas:
* **Pet Feature Validation:** Conducted deep-dive testing on pet profile customization, ensuring that visual assets (colors, patterns, eyes) persisted correctly across different game states.
* **Backlog Management:** Collaborated with the development team to prioritize the "Unleash Play" backlog, translating user feedback and bug reports into actionable QA tickets.
* **Defect Lifecycle Management:** Managed the full lifecycle of QA tickets—from initial identification and reproduction to verification and final closure.
* **UI/UX & Asset Consistency:** Verified that game assets rendered correctly across multiple platforms (iOS/Android) and that UI elements remained responsive during high-intensity interactions.

---

## 🧪 Testing Methodology
Testing for a game-centric app requires a balance between technical stability and "Fun-Factor" usability. I utilized the following strategies:

1. **Agile Ticketing Workflow:** Used a structured ticketing system to track regressions, feature requests, and critical blockers, ensuring clear communication between QA and Dev teams.
2. **State Persistence Testing:** Performed rigorous checks on the "Save/Load" logic of pet customizations to prevent data loss or visual glitches during user sessions.
3. **Exploratory & Sanity Testing:** Conducted free-form testing sessions to identify "Edge Case" bugs that automated scripts might miss, specifically in the pet interaction modules.
4. **Regression Testing:** Executed historical test suites after every Alpha/Beta build to confirm that new pet features did not negatively impact core game performance.

---

## 📂 Documentation Standards
I implemented a high standard for internal QA documentation to ensure transparency and accountability.

* **Proof of Testing (POT):** Comprehensive reports containing screen records and screenshots as visual evidence of passed/failed features.
* **QA Ticketing Templates:** Standardized the format for bug reporting, including pre-requisites, clear steps to replicate, and expected vs. actual results.
* **Sprint Summaries:** High-level reports on the status of the backlog and the overall health of the current build version.
