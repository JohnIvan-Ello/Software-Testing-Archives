# 🚛 Makati Waste Management System (MWMS): Enterprise QA
**Domain:** Smart City Logistics | ERP & Public Service Management

The Makati Waste Management System is a large-scale enterprise platform designed to manage logistical data, taxpayer records, and administrative workflows. As the QA Lead for this project, I managed extensive regression cycles to ensure system stability across multiple production releases.

---

## 🛠️ My Role & Responsibilities
I was responsible for the end-to-end quality control of the MWMS platform. My primary focus was maintaining the integrity of complex data workflows and ensuring that role-specific permissions (Admin vs. Public User) functioned correctly under high load.

### 🔍 Key QA Focus Areas:
* **Role-Based Access Control (RBAC):** Conducted rigorous security testing to verify distinct permission sets for **Tax Payer** and **Administrator** accounts.
* **Regression Testing Management:** Managed a continuous regression pipeline (spanning 2025-2026) to ensure that new logistical updates did not impact legacy billing or data modules.
* **Data Integrity & Validation:** Verified the accuracy of logistical reporting and public-facing forms to ensure government-standard data compliance.
* **Module-Based Functional Testing:** Executed deep-dive audits on critical modules, including user registration, payment processing, and administrative dashboards.

---

## 🧪 Testing Methodology
To maintain the high standards required for a city-scale management system, I utilized the following strategies:

1. **Continuous Regression Cycles:** Scheduled regular "Deep Regression" tests (Quarterly/Monthly) to monitor the health of the entire system as new features were deployed.
2. **Permission Boundary Analysis:** Used black-box testing techniques to ensure users could not access unauthorized administrative endpoints.
3. **Cross-Platform Compatibility:** Verified system responsiveness and UI consistency across different browsers and hardware configurations.
4. **Defect Lifecycle Management:** Streamlined the reporting process for logistical bugs, ensuring clear communication between the testing team and the development team.

---

## 📂 Documentation Standards
Documentation for MWMS followed a high-security protocol. I utilized standardized **Regression Templates** to track historical system performance. 

* **Proof of Testing (POT):** Comprehensive logs generated for every major deployment.
* **Admin vs. Tax Payer Test Scripts:** Specialized documentation for verifying user-specific logic.
* **Audit Trails:** Maintained detailed logs of all passed/failed cases for compliance review.
